# 比較演算子の多引数対応マイルストーン

## 目的

`lisp/stdlib.lisp`の比較演算子`>`/`=`/`<=`/`>=`/`/=`は、C側の`<`ビルトイン
（`src/lisp.c`の`lisp_builtin_lt`）から導出されているが、「CLと異なり2引数のみサポートする
（明示的な範囲の限定）」という過去の意図的な設計判断により2引数専用のままだった。

一方でC側の`<`自体は`bare_metal_lisp.md`マイルストーン29の時点から可変長引数（隣接ペアが
すべて単調増加を満たせばt）に対応済みだった。本ドキュメントは、`lisp_lambda_list_keywords.md`
マイルストーン89で導入された`&rest`ラムダリストキーワードを使い、`>`/`=`/`<=`/`>=`/`/=`の
5つをCommonLispと同じ可変長引数対応に書き換えるマイルストーンを記録する。

マイルストーン番号は既存ロードマップの最新完了マイルストーン89に続く**90**とする。

## 設計方針

### `<`はC側のまま変更不要

`lisp_builtin_lt`は`(< a b c ...)`形式の可変長引数・単調増加判定にすでに対応している
（milestone29の実装）。本マイルストーンでは変更しない。

### `>`は`<`を逆順引数で呼ぶことで導出する

`(a b c ...)`が単調減少（`a>b>c>...`）かどうかは、逆順にした列に対して`<`の単調増加判定を
かけるのと同値である。`reverse`（`lisp/compiler.lisp`、milestone65）と`apply`（C側ビルトイン、
`(apply fn args-list)`の2引数形式）はどちらも既存機能でそのまま使えるため、新規ヘルパーは
不要:

```lisp
(defun > (&rest args) (apply < (reverse args)))
```

### `<=`/`>=`/`=`は隣接ペア判定のヘルパーで導出する

等号を含む比較（`<=`/`>=`）や数値の等価性（`=`）は、いずれも「隣接ペアがすべて条件を満たす」
という判定に帰着できる（推移律が効くため、隣接ペアだけ見れば十分）。新規ヘルパー
`every-adjacent-pair`を追加し、3つとも述語を差し替えるだけで導出する。

### `/=`は全ペアの相異性が必要（推移律が効かないため隣接ペアでは不十分）

CLの`/=`は「どの2つも等しくない」という意味で、`/=`自体には推移律が効かない
（`(/= 1 2 1)`は1番目と3番目が等しいので`nil`だが、隣接ペアだけ見ると`1≠2`かつ`2≠1`で誤って
`t`になってしまう）。そのため`every-adjacent-pair`とは別に、全ペアの相異性を見る専用の
ヘルパー（`distinct-from-rest`/`all-distinct`）を追加する。

### 0引数・1引数の扱い

`every-adjacent-pair`/`all-distinct`はいずれも「比較対象のペアが無ければt」という自然な
再帰の基底ケースになるため、1引数呼び出しは常にtになる（CLでは0引数はエラーだが、本
プロジェクトは他のmilestoneでも「起こり得ない異常系まではバリデーションしない」方針の
ため、0引数も特別扱いせず同じくtを返す動作のままにする）。

### `zerop`/`plusp`/`minusp`は変更不要

既存の2引数呼び出し（`(= n 0)`等）のままで、可変長版の`=`/`<`でもそのまま動く。

## ファイル構成

新規ソースファイルは追加しない。既存の`lisp/stdlib.lisp`のみを変更する。検証用フィクスチャは
新規の`test/lisp/test-comparison-operators.lisp`を追加する。

## スコープ外として明記する項目

- CLの`/=`以外の演算子（`=`/`<=`/`>=`）を全ペア判定にする一般化。推移律が効くため隣接ペア判定
  で十分であり、あえてO(n^2)にする理由がない。
- `<`自体のC側実装の変更（milestone29で既に可変長対応済みのため対象外）。
- 数値以外の型（文字列等）への比較演算子の拡張。

## 検証方針

`make build`でクロスコンパイルが通ることと、`make test`（`test/lisp/`配下の全フィクスチャを
QEMU/OVMFヘッドレスで実行するハーネス）に既存回帰が無いことを確認する。新しいpanicシナリオは
導入していないため、`lisp_lambda_list_keywords.md`マイルストーン89のような個別QEMU対話での
panic確認は不要と判断した。
